Step 1: Encrypted CRUD with Browser Storage

Create invoices encrypted with AES-256-GCM. Data is stored in localStorage — but only as ciphertext. Open DevTools → Application → Local Storage to see the encrypted blobs.

Step 2: Multi-User Access Control

Grant access to an operator (read-write on invoices) and a viewer (read-only). Watch the viewer get blocked from writing.

Step 3: Offline → Online Sync

Two "devices" (Office & Home) each with their own local storage, sharing a simulated cloud. Work offline, then sync.

Step 4: Persist Across Reload

The browser adapter persists to localStorage. Reload the page — your data survives. This proves offline-first persistence.

Step 5: Backup & Restore

Dump the compartment as JSON (all ciphertext), download it, clear storage, then restore from the backup file.

Step 6: Audit History & Diff

Every change is tracked automatically. Browse version history, see field-level diffs, and time-travel to any past version.
